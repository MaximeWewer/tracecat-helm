Tracecat {{ .Chart.AppVersion }} deployed as release "{{ .Release.Name }}" in namespace "{{ .Release.Namespace }}".

  PREREQUISITE: the CloudNative-PG operator must be installed cluster-wide
     (this chart only ships the PostgreSQL Cluster CRs, not the operator):
     helm repo add cnpg https://cloudnative-pg.github.io/charts
     helm upgrade --install cnpg cnpg/cloudnative-pg -n cnpg-system --create-namespace

Components:
  api · ui · worker · executor · agent-worker · agent-executor{{ if .Values.litellm.enabled }} · litellm{{ end }}{{ if .Values.mcp.enabled }} · mcp{{ end }}
  + Temporal (subchart) · 2× CloudNative-PG (app/temporal) · Redis · RustFS (S3)

Bootstrapping order (no manual steps needed):
  1. CNPG provisions the app + temporal PostgreSQL clusters.
  2. The migrations Job waits for Postgres, then runs `alembic upgrade head`.
  3. Temporal schema job + the post-install temporal-setup hook register the
     namespace and search attributes.
  4. Workloads come up once their wait-for-postgres init succeeds.
